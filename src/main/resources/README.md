# Runtime Resources Directory

This directory structure (`src/main/resources/`) exists at the **project root level** (separate from `app/src/`) for a specific architectural reason.

## Purpose

**Runtime persistence for pre-computed cluster identification results.**

- These files are generated by background jobs in the GUI application
- They are packaged into the application JAR at build time
- They provide cached cluster/correlation function data to avoid expensive recomputation

## Directory Structure

```
src/main/resources/
└── cluster_data/
    ├── BCC_A2_T_bin/
    │   └── cluster_result.json
    ├── Nb-Ti_BCC_A2_T/
    │   └── cluster_result.json
    └── {modelId}/
        └── cluster_result.json
```

## How It Works

1. **Generation**: GUI jobs (via `BackgroundJobManager`) run cluster identification
2. **Storage**: Results saved to `src/main/resources/cluster_data/{modelId}/`
3. **Loading**: `ClusterDataCache` loads from classpath or filesystem at runtime
4. **Build**: Gradle packages these into the JAR's resources

## Why Separate from app/src/?

- **Build isolation**: Application source (`app/src/`) contains code; this contains data
- **Gradle convention**: Multi-module projects keep shared resources at root level
- **Version control**: Data files can be selectively committed/ignored independently
